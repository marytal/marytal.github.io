Keep Heroku Apps Awake

14 Dec 2014
I host my rails apps on Heroku, which is an awesome service.
It was becoming a problem though, that after an hour of disuse the apps would go into idle mode. When an app is idling, the time to start it up again ranges from 8 - 15 seconds, by which point many users just give up and click elsewhere.

I fixed this by installing a Heroku add-on called Scheduler. This add-on makes it possible to carry out tasks periodically. I then wrote a custom rake command that sends GET requests to all of my apps, keeping them from idling.

Lessons Learned:

  1. Difference between web and worker dynos
  2. Building custom rake tasks
  3. How to use Heroku add-ons


Now all of my Rails apps load instantaniously! Yaay!
Side Note: If you schedule too many tasks, Heroku will start charging you - so take care to monitor your usage.
